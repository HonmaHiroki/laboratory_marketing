# スキル：記事作成（SEO・LLMO対応）

**使用するエージェント：** campaign-executor（全グループ）/ content-strategist  
**入力：** 軸キーワード・市場調査データ・競合データ・CEP  
**出力：** 記事構成案・記事本文・校閲チェック結果

---

## 前提：検索エンジン環境の変化

AIエージェント・検索エンジンは「情報を求めるユーザー」と「ニーズに合うコンテンツ」をマッチングするシステム。
**ユーザーニーズに応えるコンテンツを作ることが最善のSEO・LLMO対策。**

| 観点 | 現状（2026年4月時点） |
|------|---------------------|
| 検索エンジンシェア | Google 55.39% / Bing 36.41% / Yahoo! 6.38%（Google一強ではない） |
| AI統合 | BingはOpenAIと提携・ChatGPTの標準検索バックエンドにも採用 |
| ゼロクリック化 | Google AI Overview表示時、オーガニックCTRは61%減・広告CTRは68%減 |
| AI検索の普及 | ChatGPT・Copilot・Gemini・Perplexityで検索が完結するケースが増加 |
| Googleの評価方針 | AI生成か否かは問わず「品質」と「人間が提供する価値」を評価。E-E-A-Tの「経験」の比重が増加 |

**結論：** Google・Bing・AI Overview・ChatGPT・Perplexityのどこから評価・引用されても通用する、
**一次情報・独自の経験を盛り込んだ質の高いコンテンツ**を作ることが最大の対策。

## STEP 1：ユーザーニーズ調査

### 1-1. 市場調査（4ステップ）

**① 関連キーワード抽出**

```
軸キーワードの選定 → サジェストキーワード抽出 → 関連キーワード抽出
```

- 軸キーワード：自社にとって重要かつ関連性が最も高い1語キーワード（商品名・商品ジャンル等）
- サジェストキーワード：軸キーワードと一緒に検索されるワード（ラッコキーワードで抽出）
- 関連キーワード：軸キーワードの周辺・類似キーワード（「周辺語・連想語」「類語・同義語」から抽出）

**② キーワードボリュームの把握**

- Google Keyword Plannerで検索ボリュームを把握（少額広告出稿で詳細数値が見れる）
- Ubersuggestで各キーワードの難易度をチェック

**③ キーワードグルーピング（検索クエリ4分類）**

| クエリ種別 | 別名 | 内容 | 全体に占める割合 |
|----------|------|------|--------------|
| **インフォメーショナルクエリ** | 情報型 | あるトピックを知りたい | 約8割 |
| **トランザクショナルクエリ** | 取引型 | アクションを起こしたい（顕在ニーズ） | - |
| **ナビゲーショナルクエリ** | 案内型 | 特定のページに行きたい | - |
| **ビジットインパーソンクエリ** | エリア型 | 近くの情報を探したい | - |

**インフォメーショナルクエリの8つの検索動機（バタフライ・サーキット）**

```
さぐる系（情報収集自体を楽しむ）
├── 気晴らしさせて：情報収集自体を楽しみたい
├── 学ばせて：網羅的な知識を蓄積したい
├── みんなの教えて：世間が選んでいるものを知りたい
└── にんまりさせて：知る人ぞ知るものを知りたい

かためる系（意思決定を固めたい）
├── 納得させて：今の考えが正しいか確認したい
├── 解決させて：今すぐ役立つ答えが欲しい
├── 心づもりさせて：期待値を調整しておきたい
└── 答え合わせさせて：自分の選択が正しいと思い込みたい
```

ユーザーは「さぐる」「かためる」を蝶のように行き来する（バタフライ・サーキット）。
キーワードだけでニーズが掴めない場合は実際に検索して検索エンジンの解釈を確認する。

シーズナルクエリ（季節限定の検索需要）はGoogleトレンドで需要の突出を確認する。

**④ ニーズグループのマッピング（コトラー5A）**

| フェーズ | 該当クエリ種別 | 内容 |
|---------|-------------|------|
| **認知** | インフォメーショナル / ビジットインパーソン | 商品・サービスを知るきっかけとなるカテゴリーキーワード |
| **訴求** | インフォメーショナル / ナビゲーショナル | カテゴリーに興味を持ちさらに知りたいキーワード |
| **調査** | インフォメーショナル / ナビゲーショナル | 自分に最適な情報を競合含めて検索するキーワード |
| **行動** | トランザクショナル | 購入・行動したい時に検索するキーワード |
| **推奨** | インフォメーショナル / ナビゲーショナル | 購入後の情報・他者への推奨時に検索するキーワード |

### 1-2. 競合調査（3ステップ）

**① 競合の選定**

対策キーワードで上位10位までのサイトを競合として抽出する。

**競合ポジショニングマップ（四象限）**

```
縦軸：対策キーワードの上位占有率　横軸：想定獲得検索ボリューム

第一象限（右上）：ボリューム大×占有率高 → 市場のリーダー
第二象限（左上）：ボリューム小×占有率高 → 網羅的だが質は高くないサイト
第三象限（左下）：ボリューム小×占有率低 → 特に特徴のないサイト
第四象限（右下）：ボリューム大×占有率低 → ニッチ領域特化サイト
```

**② 指標調査**

| 区分 | 指標 | 調査方法 |
|-----|------|---------|
| 内部要素 | インデックス数 | `site:ドメイン` で検索 |
| 内部要素 | コンテンツ網羅性 | `site:ドメイン キーワード` で検索 |
| 内部要素 | アクセシビリティ | Lighthouse（Chrome拡張） |
| 内部要素 | モバイルフレンドリー | モバイルフレンドリーテスト |
| 内部要素 | ページ表示速度 | PageSpeed Insights |
| 外部要素 | 被リンク数 | Moz Link Explorer / hanasakigani.jp（無料・1日3回まで） |
| 外部要素 | オーソリティ | Moz Link Explorer |
| その他 | EAT難易度 | キーワード難易度分析ツール（100点満点評価） |
| その他 | SEO難易度 | キーワード難易度分析ツール（100点満点評価） |

**③ 競合比較・自社の戦略タイプ決定**

| タイプ | 概要 | 必要リソース |
|-------|------|------------|
| **全方位型** | 質も量も求める | 専門家配置＋量産リソース |
| **量産型** | 量を求める | 量産できるリソース |
| **ニッチ型** | 質に特化し量は求めない | 質を担保する専門家配置 |
| **リーン型** | 保有リソースの範囲でスタート | 限定的なリソースで開始 |

## STEP 3：記事構成案作成

### 3-1. 基本情報整理（7項目）

```markdown
## 記事基本情報シート

【①ターゲットキーワード】
1キーワードに絞り込む。以下の優先順位で選定：
ロングテール（3語以上・最もニーズ顕在化）
  → 既に作成済みならミドルワード（2語）
  → 既に作成済みならビッグワード（1語・最大ボリューム・競合多）

キーワード深掘り手法（4つ）：
- サジェストキーワード（ミドル/ロングテール語に対して再調査）
- 再検索キーワード（SEO検索意図分析ツールで調査）
- People Also Ask（AlsoAskedで調査）
- 共起語（ラッコキーワードで調査）
- 検索結果の上位競合ページ（ラッコキーワードの見出し抽出機能で上位20記事を分析）

見出し点数化ルール：
  1位で採用されたラベル = 10ポイント
  10位で採用されたラベル = 1ポイント
  （検索順位が高いページのトピックを優遇する）

【②ターゲットユーザー】
デモグラフィック（年齢・性別・職業・学歴・家族構成）
ジオグラフィック（住所・勤務地・周辺環境）
サイコグラフィック（ライフスタイル・価値観・悩み）
※ Q&Aサイト・口コミ・SNSの実際の発言から具現化する

【③ターゲットボリューム】
Google Keyword Plannerで検索ボリュームを調査

【④前提条件】
記事を読むために最低限必要な知識・環境

【⑤検索動機】
ユーザーがその検索行動で何を解決したいか

【⑥記事読了後のアクション】
記事閲覧後のユーザーの理想的なネクストアクション

【⑦独自性】
- 新規性：他にはない強み
- 顧客メリット：ターゲットが得るメリット
- 自社メリット：自社が得るメリット
（顧客と自社のWIN-WINになっているか確認）
```

### 3-2. タイトル作成

タイトルは検索結果で最も目立つテキスト。**トリガー×要素**の組み合わせで設計する。

### トリガー（10種類・いずれか1つ選択）

| トリガー | 心理効果 | 説明 |
|---------|---------|------|
| 流行 | バンドワゴン効果 | 流行している情報への関心 |
| 同調 | バーナム効果 | 自分に当てはまると感じさせる |
| 承認 | ヴェブレン効果 | 評価されている理由を気にさせる |
| 権威 | ハロー効果 | 輝かしい実績で信頼させる |
| 限定 | スノップ効果 | 特別感・限定感で差別化欲求を刺激 |
| 禁止 | カリギュラ効果 | 禁止されるとかえって興味が湧く |
| ギャップ | 認知的不協和 | 常識とのギャップで疑問を生む |
| 損失回避 | プロスペクト理論 | 損をしないと安心させる |
| 質問 | - | 答えを知りたくさせる |
| 悩み | - | 悩みの解決を提示する |

### 要素（タイトルの魅力を増す。任意）

| 要素 | 心理効果 | 説明 |
|-----|---------|------|
| 数字 | - | ベネフィットを具体的に想像させる |
| チラ見せ | ツァイガルニク効果 | 一部しか見せず続きを読みたくさせる |
| 呼びかけ | カクテルパーティ効果 | ターゲット限定の呼びかけで自分ごと化 |

**タイトル作成チェック：**
- [ ] トリガーを1つ選択しているか
- [ ] 要素を無理なく組み込めているか（不自然なら入れない）
- [ ] ターゲットキーワードが前方に入っているか

#### A：ロジカル構成（総論・各論・結論の3部構成）

```markdown
【総論】＝ 興味付け + 結論
- 興味付け：タイトルのトリガーに応じた方針で書く（下表参照）
- 結論：記事を読むことで何がわかるかを先に簡潔に提示

| トリガー | 興味付け方針 |
|---------|-----------|
| 流行 | 流行背景・影響度を紹介 |
| 同調 | 読者に近い人・憧れの人の意見を紹介 |
| 承認 | 評価している理由を説明 |
| 権威 | 権威ある人の実績紹介 |
| 限定 | 限定情報の理由と内容を紹介 |
| 禁止 | 禁止の理由を説明 |
| 質問 | 質問の答えを記載 |
| 悩み | 悩みを解決する答えを紹介 |
| ギャップ | 矛盾が生じている理由と解消できる結論を提示 |
| 損失回避 | 現状を続けると損をする現実を伝える |

【各論】＝ 理由 + 要点 + 具体例（各パラグラフは主題文/支持文/終結文の3構成）

理由パラグラフ：
  主題文＝結論への不安・疑問点 / 支持文＝それを解消する説明 / 終結文＝主題文の答え

要点パラグラフ：
  主題文＝結論に至る重要ポイント / 支持文＝ポイント毎の詳細 / 終結文＝主題文の要約

具体例パラグラフ：
  主題文＝体験談・身近な事例 / 支持文＝事例の具体的内容 / 終結文＝事例から得た結論

【結論】＝ 総論＋各論の要約
ピーク・エンドの法則（ピークとエンドの度合いで経験を判断する心理）を活用し、
各論（ピーク）と結論（エンド）を丁寧に記述する
```

#### B：エモーショナル構成（3パターンから選択）

| パターン | 対象ユーザー | 目的 |
|---------|-----------|------|
| **興味喚起** | 既に興味を持っている | 情報提供で行動喚起 |
| **問題解消** | 悩みを抱えている | 解決策提示で行動喚起 |
| **ストーリー** | 興味も悩みも明確でない | 物語で新しい価値観を提供し行動喚起 |

各パターンとも共通構造：**導入 → 感情変容 → 正当化**

##### B-1：興味喚起

```markdown
【導入】＝ 興味付け（ポジティブトリガーのみ使用：流行/同調/承認/権威/限定/禁止/質問）
※ 悩み・損失回避はネガティブ感情のため使用しない

【感情変容】
- 利用シーン：具体的にイメージできる利用場面を紹介
- スペック：視覚・聴覚・嗅覚・触覚・味覚の5感で可能な限り伝える
- 価格・商品：値段の情報を明記
- 価値の伝え方（3種類）：
  金銭的利益（プロフィット）：金額換算した表現
  機能的利益（メリット）：機能面の優位性
  情緒的利益（ベネフィット）：機能から得られる気分

【正当化】
- 確信：感情で買って理屈で正当化する人間の心理を利用し、
  第三者評価・体験談で「買う理由」を補強する
```

##### B-2：問題解消

```markdown
【導入】＝ 気づき（ネガティブトリガー使用：悩み/ギャップ/損失回避）

【感情変容】（3ステップ）
1. あおり：問題を放置することの不利益を提示
2. 解決策提示：課題を明確化してから解決策を教える
3. ベネフィット：問題解決後に得られる未来を提示

【正当化】
- 理由・根拠：実績・体験談で解決策の有効性を補強する
```

##### B-3：ストーリー

```markdown
【紹介】＝ 主人公の設定
主人公の6条件：
□ 共感できる人物
□ 学ぶことのある人物
□ 応援したくなる人物
□ 最後に勝つ価値のある人物
□ シンプルで納得感のある動機がある人物
□ 現実に存在しうる人物

プロフィール項目：見た目／行動特徴／性格／好き嫌い／ウォンツ／ニーズ／障害
（ピクサー式エレベーターテスト：「危機にどう対処するか」でキャラクターを作り込む）

【導入】＝ 日常生活（問題意識欠如）→ 行動喚起（問題提起のきっかけ）

【感情移入】（9ステップ：ヒーローズジャーニー型）
1. 喚起に対する拒絶（変化への葛藤）
2. メンターとの出会い（転機）
3. 未来への領域へ踏み込む（一歩前進）
4. 試練・仲間・敵（経験）
5. 未知の世界へ接近（新たな景色）
6. 最大の試練（最大の壁）
7. 報酬（栄光または次への活路）
8. 帰路（変化への気持ちの高ぶり）
9. 復活・帰還（成長した姿で問題解消）
```

#### C：行動への最後の一押し（共通・全エモーショナル構成の最後に追加）

**行動喚起トリガー（4種類）**

| トリガー | 内容 |
|---------|------|
| 限定（希少性） | 数量・期間・対象限定での提供 |
| 保証 | 返金・返品保証で安心感を与える |
| 特典 | 関連性の高い特典を付与（限定と組み合わせ可） |
| 比較 | 競合・現状利用品・未来との比較で優位性を示す |

**行動しやすくするテクニック（2種類）**

| テクニック | 内容 |
|---------|------|
| **回避** | 選択肢を3個+α程度に絞る。本当に売りたいものは選択肢の真ん中に配置（極端回避性） |
| **排除** | 余計な情報・導線は視界から外し、起こしたいアクションに集中させる |

**クロスセル（行動後にさらなる行動を促す）**

| 心理効果 | 内容 | 活用例 |
|---------|------|-------|
| **テンション・リダクション効果** | 決断後の緊張緩和で気が緩む状態 | 「あと〇円で送料無料」「購入完了後に関連商品提示」 |
| **コンコルド効果** | 投資した時間・金額を無駄にしたくない心理 | ポイント失効前の利用促進・段階的コレクション商品 |

## STEP 4：記事コンテンツ作成

### 4-1. 品質の2軸

**E-E-A-T（専門知識・経験・権威性・信頼性）**

| 要素 | 内容 |
|-----|------|
| 専門知識 | その分野の知見があること |
| 経験 | 一次情報・実体験に基づく内容（比重が高まっている） |
| 権威性 | 業界での地位・実績 |
| 信頼性 | 責任者・問い合わせ先など担保情報の明記 |

**情報の網羅性と冗長性**
メインコンテンツが網羅的かつ不要な情報を冗長に書いていない状態。

### 4-2. YMYL（Your Money Your Life）領域への注意

以下に該当する場合は特に品質基準を高める：

```
□ ニュース・時事問題
□ 公民・政府・法律
□ 金融（投資・税金・保険・ローン等）
□ 買い物（商品調査・購入関連）
□ 健康・安全（医療・薬・緊急時対応）
□ 人々のグループ（国籍・人種・宗教・性別等のセンシティブ情報）
□ その他（フィットネス・住居・大学選択・就職等、人生の重要決定に関わるもの）
```

YMYL領域では記事内容だけでなく、カスタマーサポート・ユーザビリティの充実も品質担保に含まれる。

## STEP 5：校閲・監修チェックリスト（23項目）

Googleの「検索品質ガイドライン」「質の高いサイトの作成方法についてのガイダンス」準拠。

### 品質（14項目）

```
□ 1. 事実に関する誤りがない（信頼できる情報源から確認できないものは載せない）
□ 2. 物事の良い面・悪い面両方を捉えている
□ 3. 他コンテンツにはない独自性・付加価値がある
□ 4. 情報発信者（責任者）を公表している
□ 5. 不備時に対応できる環境（問い合わせ・サポート・SNS）が整っている
□ 6. 1パラグラフ1テーマが守られている
□ 7. 一文の主語→述語は1つのみで対の関係になっている
       NG: シアトル留学の経験は、自分を様々な角度から客観視できた
       OK: 私は、シアトルに留学したことで、自分を様々な角度から客観視できた
□ 8. 修飾語と被修飾語が近くに配置され対の関係になっている
       NG: 私は頻繁に結婚について考える
       OK: 私は結婚について頻繁に考える
□ 9. 文末が敬体（です・ます）か常体（だ・である）で統一されている
       例外：会話文（「」内）は敬体・箇条書きは常体が基本でも周囲は統一する
□ 10. 一文は40〜60文字を目安にしている
□ 11. 難しい漢字・カタカナ語を多用していない
□ 12. 箇条書きは4±1項目（マジカルナンバー）に収まっている
       ※「多すぎて覚えられない」という印象を与えたい場合は例外で多めもOK
□ 13. 時系列に左右された表現を用いない
       NG: 昨夏は猛暑だった　OK: 2018年の夏は猛暑だった
□ 14. 二重表現・二重否定を使わない
       二重表現例: あとで後悔する、必ず必要だ
       二重否定例: 少なくない、思わなくない
□ 15. 全角・半角どちらかに統一されている
```

### ユーザビリティ（4項目）

```
□ 16. モバイルユーザーに最適なユーザビリティが提供されている
□ 17. コンテンツ量が多い場合は目次を設置している
□ 18. 文字サイズ・行間・余白で読みやすくしている（近接・整列・反復・対比を意識）
□ 19. テキストで伝えにくい内容は画像・動画を使用している
```

### 需要（4項目）

```
□ 20. ターゲットキーワードの情報が網羅され、不要な情報は掲載していない
□ 21. ターゲットキーワードがタイトルの前方に入っている
□ 22. タイトルと導入文だけでコンテンツ内容を理解できる
□ 23. 見出しの中にターゲットキーワードが含まれている
```
