# スキル：インサイト構造化フォーマット

**使用するエージェント：** market-scout / insight-analyst  
**入力：** 生のSNS口コミ・アンケート自由回答・N1インタビューメモ  
**出力：** インサイト構造（シーン/ドライバー/エモーション/バックグラウンド）

---

## インサイトとニーズの区別

| | インサイト | ニーズ |
|--|-----------|-------|
| 状態 | 本人も気づいていない隠れた動機 | 本人が認識できている顕在欲求 |
| 競合状況 | 競合他社も知らない → 競争優位の源泉 | 大半の競合が対処済み |

**ニーズを深掘りしてインサイトを発見する。**

---

## 構造化の手順

### STEP 1：生データを収集する

口コミ・アンケート・インタビューから「感情が含まれる発言」を抽出する。
感情語（嬉しい・困った・なぜか・ついつい・気づいたら など）をフックにする。

### STEP 2：4要素に分解する

```
【シーン】
どんな状況・場面・行動・状態で感情が生まれたか
（いつ・どこで・誰と・何をしているとき）

【ドライバー】
感情を生み出した直接的な要因
（商品の物性・機能・情報・他者の言動・環境など）

【エモーション】
生まれた気分・気持ち・感情
（嬉しい・悔しい・安心した・驚いた・なぜか気になる など）

【バックグラウンド】
その感情が「価値」または「不満」である背景的な理由
（その人の価値観・過去経験・社会的文脈・自分ルール）
```

### STEP 3：インサイト文を生成する

```
「[シーン]のとき、[ドライバー]によって[エモーション]が生まれる。
それは[バックグラウンド]という背景があるためだ。」
```

### STEP 4：信頼度を付与する

| 信頼度 | 条件 |
|-------|------|
| 高 | 複数の独立したソースで同じ構造が確認できる |
| 中 | 1つのソースで明確に観察できる |
| 低 | 間接的・推測的。要追加調査 |

### STEP 5：アウトプット

```markdown
## インサイト：[タイトル]

- **シーン**：
- **ドライバー**：
- **エモーション**：
- **バックグラウンド**：
- **インサイト文**：
- **対象顧客タイプ**：ロイヤル / 競合顧客 / 未顧客
- **信頼度**：高 / 中 / 低
- **ソース**：[収集API・調査名・取得日]
```
