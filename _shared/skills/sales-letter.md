# スキル：セールスレター設計

**使用するエージェント：** campaign-executor（全グループ）/ content-strategist  
**入力：** 商品・サービスの特徴・ターゲット情報・CEP・仮説  
**出力：** セールスレター全文（ヘッドライン〜P.S.）

---

## セールスレターの全体構造

セールスレターはトップ・ボディ・クロージングの3領域で構成される。

```
【トップ】
  ヘッドライン  ← 最重要。見込み客の目を止めさせて本文を読ませる
  オープニング  ← ヘッドラインの続き。さらに深く引き込む

【ボディ】
  ベネフィット  ← 商品で得られる結果・変化（特徴ではない）
  ブレット     ← ベネフィットの箇条書き（好奇心を刺激）
  信頼性      ← 顧客の声・実績・保証・ストーリー
  オファー    ← 価格・特典・保証・期限

【クロージング】
  行動喚起    ← 今すぐ購入しないといけない理由

【P.S.】    ← ヘッドラインの次に読まれる重要箇所
```

## STEP 1：ベネフィット整理

### 特徴→ベネフィット変換テンプレート

```
この商品は【特徴】があります。
だから【ベネフィット】ができるようになります。
```

### ベネフィットを考える3つの軸

| 軸 | 内容 |
|---|------|
| **時間（Time）** | より早く・時間が早くなる・早くできる |
| **効果（Effort）** | より確実に・安定して効果が出る |
| **お金（Money）** | より安く・お得・無駄な投資をしない |

### ベネフィット作成5ステップ

```
STEP1：特徴をリストアップする（成分・機能）
STEP2：特徴が存在する理由を考える（なぜその機能があるのか）
STEP3：特徴をベネフィットに転換する（「〜できます」「〜のようになります」）
STEP4：ベネフィットが実現するイメージを伝える（お客さんはどうなるか）
STEP5：ベネフィットを広げる（感情的・時間的変化）
```

### USP（独自のセールスポジション）の定義

最大の効果を狙うには **1つのベネフィットだけを強調** する。

USPの3条件：
1. **独自性があるように見える**（完全独自でなくてもよい）
2. **役に立つ特徴である**（独自性より実用性を優先）
3. **コンセプトが単純である**（説明が難しいものは売れない）

## STEP 2：アプローチパターンの選択

ターゲットの認知度に応じてアプローチを変える。

| 認知度 | 無関心 | 問題認識 | 解決認識 | 商品認識 | マニア |
|-------|-------|---------|---------|---------|-------|
| 商品訴求 | × | × | △ | ○ | ◎ |
| 問題訴求 | × | ○ | ○ | ○ | ○ |
| アプローチ | 間接 | 間接/問題解決 | 問題解決 | USP | オファー |

### アプローチ別フレームワーク

**A：間接アプローチ（無関心〜問題認識層）**

「セールスレターを読む事を売る」。直接商品を押し付けない。

- **QUESTフレームワーク**
  1. Qualify（選別）：ターゲットを絞り込む
  2. Understand（共感）：悩みに共感する
  3. Education（教育）：新しい視点を提供
  4. Stimulate（欲求の刺激）：欲しいと思わせる
  5. Transition（行動の呼びかけ）：行動を促す

- **欲求充足モデル**（今すぐ必要ではないが欲しい層）
  1. 共感 → 2. 信用 → 3. 想像 → 4. 比較 → 5. 不安除去 → 6. 買う理由 → 7. 行動

- **潜在的需要モデル**（必要性に気づいていない層）
  1. 認知的不協和 → 2. 必要性 → 3. 信用① → 4. 解決方法 → 5. 信用② → 6. 比較 → 7. 不安除去 → 8. 買う理由 → 9. 行動

**B：問題解決アプローチ（問題認識〜解決認識層）**

- **PASONAの法則**
  1. Problem（問題を提起）
  2. Agitation（問題を炙り出して煽る）
  3. Solution（解決策を提示）
  4. Narrow down（限定感を出す）
  5. Action（購買行動につなげる）

- **問題解決モデル**
  1. 期待（LPを見て解決できそうと期待させる）
  2. 共感（「この問題ですか？」と質問形式で確認）
  3. 解決方法（商品情報の詳細）
  4. 比較（競合との比較）
  5. 信用（実績・証拠）
  6. 不安（購入直前の不安を取り除く）
  7. 感情（想い・物語で差別化）
  8. 行動

**C：USPアプローチ（商品認識層）**

- 「私に〇日をください。あなたを【ベネフィット】してみせます」
- 「もしあなたが（条件）なら、【ベネフィット】できます」
- 「【ベネフィット】が欲しい人は他にいませんか」

## STEP 3：ヘッドライン生成

ヘッドラインはセールスレター最重要箇所。目的は「本文を読ませること」。

### ヘッドラインの基本アイディア（6種）

1. 最も大きな約束・メリットをオファーする
2. 理想の見込み客を明確にし群衆から選び出す（「注意！〜で困っている方へ」）
3. ニュースを提供する（「最新の〜は〜」）
4. 即効・簡単なオファーをする（質問系にすると疑われにくい）
5. 挑戦的な質問をする（YES/NOで答えられない質問）
6. 続きを読みたくさせる（意外性・秘密・フック）

### ヘッドラインフォーミュラ（選択して使用）

| フォーミュラ | テンプレート | 例 |
|-----------|-----------|---|
| 〜する方法 | （メリット）する方法 | 税金を減らして、お金を貯める方法 |
| バーゲン | 商品が、たったの〇〇で | 12枚のCDセットがたった1枚の値段で |
| ストーリー | どのように【意外なこと】が【メリット】を実現したか | どのように一発芸が私をトップ営業マンにしたか |
| 状況に応じた質問 | 【状況】なら【ベネフィット・理由】 | 妊娠中？トウモロコシについて知っておくべきことがあります |
| 約束 | できなければ私に（ペナルティ） | 〜できなければ、このレッスンは無料。さらに25$払います |
| 顧客の声 | （顧客が語る驚きの結果） | 私はこのマーケターに昨年6,000万円支払いました。なぜなら2億円稼げたから |
| 痛みを無しに | （メリット）たとえ（面倒なこと）をしなかったとしても | 〜できます。たとえ書かなかったとしても |
| もし〜なら | もしあなたが（条件）なら（メリット） | もしあなたが安全運転ドライバーなら保険料を節約できます |
| 他にいませんか | （メリット）を手に入れたい人は他にいませんか | 映画スターみたいなルックスになりたい人は他にいませんか |
| ついに | ついに（待ち望んでいたもの）ができました | ついに、めまいを科学的に抑える方法ができました |
| 私に〇〇してください | 私に（時間）ください。（ベネフィット）にできることを証明してみせます | 7日間くれますか？あなたを新しい男に変えてみせます |
| 探しています | （やりたいこと）をする人を探しています | 子供用の本を書く人を探しています |
| なぜ | なぜ（答えが聞きたくなる挑発的な質問） | 世界最高値の雑誌。なぜ4万人のビジネスマンが毎月買うのか |
| 警告 | 警告！（何かをする）前に（これをやれ） | 警告！これを読む前にサプリを買おうなんて考えてはいけません |

### ヘッドライン強化チェック

- [ ] 保証を入れる（30日間の完全保証など）
- [ ] 具体的にする（「名詞」と「数字」を使う）
- [ ] 質問にする（困ったら疑問形に変換）
- [ ] 小学4年生が読める漢字・言葉を使う
- [ ] 好奇心を加える（続きを読みたいと思わせる）
- [ ] 主張を強化または弱める（市場の逆を行く場合も有効）

## STEP 4：オープニング生成

目的：ヘッドラインの続き。見込み客をさらに深くセールスコピーに引き込む。
ポイント：読み手に頷いてもらうこと。

### オープニングフォーミュラ（共有系）

| タイプ | 内容 | 使用例 |
|-------|------|-------|
| プレビュー | セールスレターの内容を単刀直入に伝える | 「私はこれから〜を公開します」 |
| ストーリー | ストーリーで始める | 「25年前の春の夕暮れ時、二人の若者が...」 |
| ニュース | シンプルに相手にメリットがあることを言う | 「おめでとうございます！良いニュースです！」 |
| 問題報告 | 失敗や事件を使ったオープニング | 「今、税金のことでトラブっています。なので...」 |
| 代行報告 | 「あなたの代わりに〇〇をします」 | 「12ヶ月の残業の結果、2000の秘訣を集められました」 |
| 自己紹介 | 書き手の正直な自己紹介 | 「私の名前は〇〇です。広告のプロではありません。しかし...」 |
| 感情 | 強い感情で始める | 「めちゃくちゃ頭にきています！」 |
| 限定・招待状 | 選ばれた人に対して送ることを伝える | 「この個人的な招待状は、ほんの少しの人の元にしか届いていません」 |
| 引用 | 権威ある人物の発言を引用 | 「〜というマーケティングの天才はかつてこう言いました」 |

### オープニングフォーミュラ（質問・仮定系）

| タイプ | 内容 |
|-------|------|
| YES/NOで答えられない質問 | 「12ヶ月ごとに50%の利益が出たら、あなたはどれくらい嬉しいですか？」 |
| もし〜なら | 「もしあなたが将来のインフレを心配しているなら、真剣に考えたほうがいいアイディアがあります」 |
| お願い | 「私のお願いを聞いてもらえる代わりに、この無料プレゼントを受け取ってもらえませんか？」 |

## STEP 5：ボディコピー生成

### ブレット（ベネフィット箇条書き）のテンプレート

| タイプ | テンプレート |
|-------|-----------|
| 〜する方法 | 【ベネフィット】を得る方法 / すぐに、簡単にできてしまう【ベネフィット】を得る方法 |
| 秘密・秘訣 | 【ベネフィット】を得る秘密 / 〜する秘訣 |
| 禁止・警告 | 【行動する時】、絶対にやってはいけないこと / 警告！【〜しないでください】 |
| 質問・仮定 | 【見込み客が興味を持つ質問】なら、〇〇で【ベネフィット】を得ることができます |

### 信頼性を上げる11の方法

1. **顧客の声・レビュー**（最もパワフル）
2. **Before-After / ケーススタディ**（購入前後の変化を具体的に）
3. **業界エキスパートからの推薦**（著名人・芸能人）
4. **会社情報・取引先**（所属団体・クライアント企業名）
5. **PR・新聞・雑誌記事**（「いつ」より「どこに」取材されたかが重要）
6. **実績**（資格・会員数・販売数・受賞歴・社歴）
7. **保証**（返金保証・成果保証）
8. **弱点・欠点を認める**（誠実さがガードを下げる）
9. **ストーリー**（開発秘話・CEOメッセージ）
10. **理由**（なぜこの商品か・なぜ今か・なぜあなたから買うのか）
11. **事実・調査結果・統計**（信頼性の高い出典から）

証拠の見せ方：**ビデオ > 写真 > テキスト**の順で効果が高い。

### オファーの構成

| 要素 | 内容 |
|-----|------|
| **価格** | 割引・100円オファー・半額・〇〇円引き |
| **無料** | 無料お試し・無料レポート・無料デモ・送料無料 |
| **支払い** | 後払い・分割払い・決済方法の拡充 |
| **保証** | 返金保証・2倍返金保証・アクションテイカー型返金保証 |
| **販売促進** | デラックスオファー・松竹梅・バンドル・コンテスト |
| **期限・限定** | スタートメンバー・β版割引・期間限定価格・数量限定 |
| **特典** | プレミアム（景品）・プライス（割引）・サンプリング |

**オファー5つのポイント：**
1. 期限を必ずつける
2. 相手のリスクをなくす（むしろ逆転させる）
3. ボーナス・特典をつける
4. 競合を調べておく（競合の批判は気にしない）
5. Noと言えないオファーを作る

## STEP 6：クロージング生成

目的：今すぐ行動させる。主要メリットを再提示し今すぐ注文させる。

### クロージングフォーミュラ

**A：限定性**
```
今すぐ申込みをしてください。そうすれば、【特典リスト】が無料で受け取れます。
これらが全てついて、◯◯円です。【希少性・ベネフィットのアピール】
今すぐお申し込みください。
```

**B：クロスロード（2つの道）**
```
あなたには2つの道があります。
1つめ：自分ひとりでやる道。【デメリット・苦労を描写】
2つめ：私たちと一緒にやる道。【ベネフィット・早い結果・確実性をアピール】
あなたはどちらを選びますか？
```

**C：未来像（ポジティブ/ネガティブ）**
- ポジティブ：商品を使った後の理想の未来を描写
- ネガティブ：「もし〇〇がなかったら？」と失った未来を描写

**D：テイクアウェイセリング**
```
【ベネフィット1】【ベネフィット2】【ベネフィット3】
でも、あなたは手に入れられないかもしれません。
なぜなら【あなた向きではない理由】…しかし、その弱点は強みでもあります。
```

## STEP 7：P.S.生成

ヘッドラインの次に読まれる。必ず入れること。

### P.S.フォーミュラ

| タイプ | テンプレート |
|-------|-----------|
| 行動刺激 | 「忘れないでください。この商品を買えば【ベネフィット】が得られます。これは【希少性アピール】です。今すぐ申込みしてください。」 |
| ボーナス繰り返し | 「特典は早く申し込みした方へのプレゼントです。今すぐ申込みしてください。」 |
| 保証繰り返し | 「この商品には【返金保証】があります。もし満足できなければ、何も聞かず全額返金します。」 |
| 新しいボーナス | 「今すぐ申込みしてくれたなら、上記特典に加えて〇〇を無料でプレゼントします。」 |
| 顧客の声紹介 | 「発表してほんの少しの時間しか経っていませんが、すでにこのような声をもらっています。」 |
| デッドライン強調 | 「申込みできるのは◯日までです。期日までにお返事いただければ【特典A】【特典B】はあなたのものです。」 |

## STEP 8：ターゲット別レターテンプレートの選択

| ターゲット | WHO | コスト | 売上 | 推奨テンプレート |
|---------|-----|-------|------|--------------|
| **新規集客** | 見込み客 | △ | ◎ | スターストーリーソリューション・2人の男（女）・こんな間違い・他にいませんか・ブライブレター |
| **売上UP** | 既存客 | ○ | ◎ | 理由+割引・写真+割引・お願いレター・招待状 |
| **離反復活** | 休眠客 | ◎ | ○ | 休眠客3通テンプレート（1通目：金券/2通目：念押し/3通目：最後） |

### 見込み客向け主要テンプレート

**スターストーリーソリューション**
```
1. 「私が【商品に関連すること】をすると周りのみんなは笑った。でも実際には...」
2. バカにされていた状況を描写
3. ヒーローになれた状況を描写
4. ヒーローになれたのは商品のおかげ
5. USPに基づくベネフィット
6. クロージング
```

**2人の男（女）**
```
1. 同じような境遇にある2人を登場させる
2. 同窓会で再会（相変わらず似ている境遇）
3. しかし違いもありました。1人は【うまくいっていない】もう1人は【成功している】
4. 何が違いを生じさせたのか？
5. その違いは【商品・サービス購入への第一歩となる事実】から生じた
6. USPに基づくベネフィット→オファー→クロージング
```

**こんな間違い**
```
1. 「あなたは【テーマ】でこんな間違いはしていませんか？」
2. よくある間違いや意外な事実を提示
3. 問題を深刻化させる
4. 問題が起きる原因を特定
5. 解決策として商品を紹介
6. クロージング
```

**休眠客3通テンプレート**
```
1通目：「最近来ていただけないから」1,000円分金券プレゼント
2通目：「届いてなかったかもしれません」再送・金券同封
3通目：「これで最後にします」最後の金券・これ以上の予算がないことを正直に
```

## STEP 9：推敲チェックリスト

**実施すること：**
- [ ] 1日置いてから見直す（冷静な目で確認）
- [ ] 声に出して読み上げる（読みにくい箇所を発見）
- [ ] 他人に読んでもらう（詰まる箇所を特定）
- [ ] 業界用語・難しい言葉をチェック

**キャッチコピー確認：**
- [ ] ターゲットは絞り込まれているか
- [ ] ベネフィットを強調しているか
- [ ] 感情や興味をそそる内容になっているか
- [ ] 素早く簡単に手に入りそうと感じるか

**ボディ確認：**
- [ ] キャッチコピーの内容と乖離していないか
- [ ] 難しすぎる言葉を多用していないか
- [ ] 商品・サービスで得られるメリットが記載されているか
- [ ] 結果を得られる論理的な理由・データがあるか
- [ ] 自分にも効果があるという安心感・信頼感があるか

**クロージング確認：**
- [ ] 競合より魅力的なオファーか
- [ ] 今すぐ購入しないといけない理由があるか
