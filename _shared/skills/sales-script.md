# スキル：セールススクリプト設計

**使用するエージェント：** campaign-executor（全グループ）/ content-strategist  
**入力：** 商品・サービス情報・見込み客情報・CEP・提案コンセプト  
**出力：** セールススクリプト全文（個人的な物語〜成約まで）

---

## 前提：脳の構造と意思決定の仕組み

セールスは「感じる脳・本能的な脳で始まり、考える脳で承認される」プロセス。
この順序を理解してスクリプトを設計する。

| 脳の部位 | 機能 | セールスへの影響 |
|---------|------|---------------|
| **大脳新皮質**（外側） | 理性・論理・データ処理 | 事実やデータを評価・判断する |
| **大脳辺縁系**（中間） | 感情・好き嫌い・行動決定 | 「買う・買わない」を決める感情の座 |
| **大脳基底核**（内側） | 無意識・本能・生存反応 | 変化への抵抗・現状維持本能 |

**重要原則：**
- 感情で決め、後から論理で正当化する
- 大脳新皮質の動きを止め、感じる脳と本能的な脳を活性化させる
- 制限時間は約15分（運命の瞬間の演出時間）

---

## 脳内物質の活用

| 物質 | 役割 | 活性化する方法 |
|-----|------|-------------|
| **オキシトシン**（信頼物質） | 恐怖・不安を和らげ信頼感を高める | 弱みを見せる・なぜ物語を語る・共感する |
| **コルチゾール**（ストレスホルモン） | 変化への抵抗・自衛本能を発動させる | 現状維持の批判・押し付け・情報過多 |
| **ミラーニューロン** | 共感・相手の経験を脳内で再現する | 感情移入できる物語を語る |

**狙うべき状態：コルチゾールを下げ、オキシトシンを上げる**

---

## 変化の6つの障害（スクリプトで必ず対処）

見込み客が変化を拒む6つの心理的障害を理解し、スクリプトに対処を組み込む。

| 障害 | 内容 | スクリプトでの対処 |
|-----|------|----------------|
| **①不安の雪崩効果** | 未知への恐怖で殻に閉じこもる | 物語で安心感を先に与える |
| **②孤立による不安** | 一人で変化のリスクを取ることへの恐怖 | 「多くの人が同じ状況で成功している」事例を提示 |
| **③損失の恐怖** | 今あるものを失う恐怖（利益の2倍の強さ） | 「現状維持の場合に失うもの」を先に語る |
| **④選択肢過多** | 多すぎる選択肢でコルチゾールが増加 | 提案は1つに絞る・段階的に進める |
| **⑤欠乏症候群** | 「準備が整っていない」という感覚 | 成功に必要なものが揃っていることを証明 |
| **⑥安全地帯効果** | 商談後に現状維持に戻る | 商談中に十分な危機感と変化の理由を与える |

---

## 6つの認知バイアスの活用

| バイアス | 内容 | 活用方法 |
|---------|------|---------|
| **プロスペクト理論（損失回避）** | 損失の恐怖は利益への期待の2倍 | 現状維持で失うものを「数値化」して先に語る |
| **アンカリング効果** | 最初に見た数字が基準になる | 最初に大きな数字（問題のコスト）を提示する |
| **選択支持バイアス** | 自分の過去の選択を正当化したい | 「あなたの選択は正しかった」から始める |
| **確証バイアス** | 自分の信念を支持する情報を好む | 見込み客の信念と一致する事例・データを使う |
| **利用可能性ヒューリスティック** | 思い出しやすい情報を重視する | 物語・映像・具体的な数字で記憶に残す |
| **バンドワゴン効果** | 集団に属したいという本能 | 「多くの人がすでに選んでいる」社会的証明を提示 |

## STEP 1：事前準備（商談前に必ず完了）

商談前に以下5つの問いに答えられなければ成約は望めない。

```markdown
## 事前準備チェックシート

1. 相手はどんな立場か（役職・責任範囲・意思決定権）

2. その立場における優先目標を5つ挙げると何か
   - 目標1：
   - 目標2：
   - 目標3：
   - 目標4：
   - 目標5：

3. 相手はその目標をどう数値化しているか
   （KPI・指標・達成率など）

4. その目標を達成する際の課題・問題は何か

5. 自社の製品・サービスでそれが解決できるか
   → Yes の場合：どのように解決できるか
   → No の場合：何を修正すれば解決できるか
```

## STEP 2：コンセプト整理

### 提案の目的の定義

```markdown
## 提案コンセプトシート

【誰に提案するか】

【何を提案するのか】

【WHAT（提案の手段）】
- 手段は何か：
- 手段を一言で表すと：

【顧客メリット】
- 行動メリット（何ができるようになるか）：
- 数字メリット（どれくらい改善するか）：

【ビッグアイディア】= WHAT × 顧客メリット
（以下テンプレートから選択して作成）
```

### ビッグアイディアテンプレート（7パターン）

```
「より良い<メリット>を築きたいなら、今こそ<WHAT>」
「事業を拡大し、<メリット>の関心を引くには<WHAT>に注力すべし」
「<メリット>を届けるには<WHAT>を作る必要がある」
「<メリット>の達成には、<WHAT>への投資を増やすこと」
「<メリット>の改善には<WHAT>から」
「<メリット>に集中して<WHAT>を届けよう」
「全体としての<メリット>の実現は<WHAT>が鍵になる」
```

## STEP 3：ストーリー設計

### 構成要素の整理

```markdown
## ストーリー設計シート

【解決策】
- 提案内容：
- 具体的な対応事項：
- 成功事例・実績：

【衝突（センターピン）】
- 現状引き起こしている問題と発生要因：
- 提案したい解決策と関連しそうな問題：
- 問題の発生要因の深掘り：
- クリティカルな課題（衝突）を一言で：
- 衝突を強化する事例・調査結果：

【登場人物】
- 舞台の主役（提案サービスの使い手）：
- 登場人物の現状：

【舞台設定】
- テーマ（何について話をするのか）：
- 症状（現状の数字）：
- テーマ×症状のセリフ：
  「<テーマ>について、<現状の数字>という状況にあります。」
```

### ストーリーの3部構成

```markdown
【WHY】（なぜこの話をするのか）
「<舞台設定>という状況があり、<登場人物>は、その中にいます。
本当の問題は<衝突>が起きていることです。」

【WHAT】（何が解決策か）
「その衝突を解決するには、<コンセプト>という考え方があります。」

【HOW】（どうやって解決するか）
「<解決策>によって、あなたは課題を解決できます。」
```

## STEP 4：4つの物語の設計

商談は以下4つの物語を順番に語ることで構成される。

---

### 物語①：個人的な物語（最初に語る）

**目的：** 人としての信頼を高め、見込み客に防御シールドをおろしてもらう

**伝え方の手順：**

```markdown
【導入】
「本日の本題に入る前に、2分だけ、私がなぜ今の仕事をしているかご説明いたします」

【場面転換】
聞き手を物語の舞台に連れていく
（時間・場所・状況を具体的に描写）

【物語本体】
あなたの「なぜ」を語る

【橋渡し】
あなたの「なぜ物語」が聞き手とどのような関係があるか論理的に説明する

【まとめ】
「ここで〇〇様の「なぜ」もお聞かせいただけますか？」
```

**物語に追加する3要素：**

| 要素 | 内容 |
|-----|------|
| **変化は外部から** | 語り手の素晴らしさではなく、外部からもたらされた変化として語る |
| **信念を中心に** | 物語は語り手ではなく聞き手についてのもの。あらゆる人に共感される信念を示す |
| **視覚化情報** | 信念に視覚情報を添え、聞き手が頭の中でその場面を再現できるようにする |

---

### 物語②：見込み客の物語（目標を共通認識にする）

**目的：** 相手の世界・目標を理解していることを示す。人としての信頼をさらに高める

**質問設計のポイント：**

```markdown
【やってはいけないこと】
× 誘導質問・ペインポイントを最初に突く
→ 見込み客の守勢本能が発動し、コルチゾールが増加する

【やること】
○ 成功体験・得意なことを先に聞く（ポジティブな脳内物質を促す）
○ それから苦労していることを聞く
○ 相手の「なぜ」を聞く（実績・経験ではなく信念を聞く）
○ 最後に第三者からの問題の物語で締める
```

**質問の流れテンプレート：**

```markdown
Q1（成功体験）：「最近、特にうまくいっていることを教えていただけますか？」
Q2（目標確認）：「〇〇様が今最も重要視されている目標は何でしょうか？」
Q3（数値確認）：「その目標を達成した際、どのように数値で把握されていますか？」
Q4（課題確認）：「その目標を達成するうえで、現在どのような課題がありますか？」
Q5（数値化）：「その課題によって、現在どのくらいのコストが発生していますか？」
Q6（なぜ）：「〇〇様がその目標に取り組んでいる背景には、どのような思いがありますか？」
```

---

### 物語③：問題の物語（敵役・問題の深刻化）

**目的：** 目標達成を阻んでいる問題を明確にし、危機感を数値で伝える

**物語に入れる4要素：**

```markdown
【問題】：優先すべき問題は何か
【影響力】：問題の代償を数値化（コスト・機会損失・時間損失）
【波及力】：組織全体・市場全体など大きな視野で見た影響
【障害】：これまで問題の解決を阻んできたもの
```

**数値化のフレームワーク：**

```
損失のコスト = （問題が起きる頻度）×（1回あたりの損失額） × 12ヶ月
機会損失 = （理想の状態の売上）-（現状の売上）
時間損失 = （問題対応にかかる時間）×（時給換算）× 年間発生回数
```

**第三者視点の活用：**
問題の深刻さを指摘する際は第三者（業界調査・他社事例・研究データ）の視点を盛り込む。

```markdown
「実は業界全体で見ると、〇〇という問題が〜〜%の企業で発生しています（出典：〇〇）」
「私どものお客様でも同様の問題を抱えていた〇〇社では、
 対処しないまま半年で〇〇万円の損失が発生しました」
```

---

### 物語④：製品と解決策の物語（なぜ自社だけが解決できるか）

**目的：** 見込み客の問題を解決できるのは自社だけであることを示す

**構成：**

```markdown
【証明の物語】
似たような状況の顧客が製品・サービスを使って成果を上げた話
→ コルチゾールが減少し、変化のリスクへの恐怖が弱まる

【Before-After形式】
Before：<見込み客と似た状況の顧客>は以前、<問題>という状態でした
After ：<解決策>を導入した結果、<数値的な成果>が達成できました
期間  ：<〇ヶ月>で実現しました
```

## STEP 5：変化の障害への対処スクリプト

商談中に見込み客から抵抗が生まれた際の対処手順：

### 障害の種類の特定

```markdown
「少し確認させてください。現時点でご懸念がある点を教えていただけますか？」

→ 回答を受けて以下の6種類から特定する：
  □ 信頼が足りない（人間関係の問題）
  □ 危機感が足りない（問題の深刻さが伝わっていない）
  □ 予算が足りない（または口実）
  □ 価値がわかっていない（ROIが不明確）
  □ 主張を信じていない（証拠不足）
  □ タイミングが良くないと感じている（今すぐやる理由がない）
```

### 障害種別の対処スクリプト

**信頼が足りない場合：**
```
「おっしゃる通りです。私どもとのお付き合いはまだ日が浅く、
 実績をご覧いただく機会がなかったかもしれません。
 同様の状況だった〇〇社の事例をご紹介させていただけますか？」
```

**危機感が足りない場合：**
```
「〇〇様がおっしゃる通り、緊急性が高くないかもしれません。
 ただ、先ほどお聞きした〇〇という課題を、
 このまま半年放置した場合の試算をご覧いただけますか？」
（数値化した損失を提示する）
```

**価値がわかっていない場合：**
```
「導入コストに対して得られる成果のイメージが
 まだつかみにくいということでしょうか？
 〇〇社での導入から〇ヶ月後のROIをお見せします」
```

**タイミングが良くない場合：**
```
「状況が落ち着いてから、という判断は十分理解できます。
 ただ、その間に〇〇万円の機会損失が発生し続けることになります。
 先ほどご確認いただいた数値を踏まえると、
 今すぐ着手する場合とそうでない場合の差はこれほどになります」
```

**本能を抑える原則：**
- 相手の抵抗を論破しようとしない
- 事実を並べ立てて打ち負かそうとしない
- 商談の初めに共通認識とした「相手の目標」を中心に話を組み立て直す

## STEP 6：商談スクリプトの全体構成

```markdown
## 商談スクリプト テンプレート

【PHASE 1：アイスブレイク・物語①（個人的な物語）】約3分
目的：オキシトシンを分泌させ、コルチゾールを下げる

「本日はお時間をいただきありがとうございます。
 本題に入る前に、2分だけ私がなぜこの仕事をしているかお話しさせてください」

→ 個人的な「なぜ物語」を語る（STEP 4：物語①を参照）
→ 「〇〇様はなぜこの仕事をされているのでしょうか？」と問いかける

【PHASE 2：物語②（見込み客の物語）】約4分
目的：目標と現状を共通認識にする

→ 成功体験から始め、目標→課題→数値化の順で質問
（STEP 4：物語②の質問テンプレートを使用）

「整理させてください。〇〇様は〈目標〉を実現しようとされていて、
 現状〈課題〉という状況にある。そのコストは〈数値〉ほどになっている。
 この認識でよろしいでしょうか？」（合意を確認）

【PHASE 3：物語③（問題の物語）】約3分
目的：危機感を高め、変化の必要性を感じてもらう

→ 舞台設定（テーマ×症状）から入る
→ 問題→影響力→波及力→障害の順で語る
→ 第三者データ・事例で客観性を加える
→ 損失を数値化する

「このまま現状を続けた場合、〈数値〉の損失が発生し続けることになります。
 この問題を解決することを今の最優先課題にされる理由がおわかりいただけますか？」

【PHASE 4：物語④（解決策の物語）】約3分
目的：自社だけが解決できることを示す

→ 見込み客と似た状況の証明の物語（Before-After）を語る
→ ビッグアイディアを提示
→ 具体的な解決策（HOW）を説明

「〇〇社と同様の状況にあった〇〇社では、〈解決策〉によって
 〇ヶ月で〈成果の数値〉を実現しました」

【PHASE 5：変化の障害への対処】
→ 抵抗が生まれた場合はSTEP 5の対処スクリプトを使用
→ 相手の目標を中心に話を組み立て直す

【PHASE 6：クロージング】
「〇〇様の目標〈目標〉を実現するために、
 〈解決策〉を〈時期〉から開始することについて、
 次のステップに進めますか？」
```

## STEP 7：物語の記憶定着化

人間の脳は情報の伝え方によって記憶保持率が変わる：

| 情報の伝え方 | 記憶保持率 |
|-----------|---------|
| 事実・データのみ | 5〜10% |
| 視覚資料付き | 20〜25% |
| **物語仕立て** | **65〜70%** |
| 物語×事実の組み合わせ | 事実のみの**22倍**記憶される |

**物語の基本要素（毎回チェック）：**
```
□ 設定（いつ・どこで・どんな状況か）
□ 主人公（善い人：目標・望み・夢を持つ）
□ 敵役（悪い人：主人公の望みを邪魔する）
□ 賢人（主人公に進むべき道を教える）
□ 選択と行動（主人公が決断する瞬間）
□ 結末と教訓（成功か失敗かと学び）
```

## STEP 8：スクリプト品質チェックリスト

**物語の確認：**
- [ ] 個人的な「なぜ物語」が準備できているか
- [ ] 見込み客の目標を数値で確認する質問が準備できているか
- [ ] 問題の損失を数値化できているか（金額・時間・機会損失）
- [ ] 証明の物語（Before-After）が1つ以上準備できているか
- [ ] 物語に設定・主人公・敵役・賢人・結末が含まれているか

**脳科学的確認：**
- [ ] 誘導質問・ペインポイント直撃から始めていないか
- [ ] 成功体験→課題の順番で質問しているか
- [ ] 損失（プロスペクト理論）を利益より先に語っているか
- [ ] 社会的証明（バンドワゴン効果）が入っているか
- [ ] 選択肢は1つに絞れているか（選択肢過多を避ける）
- [ ] 第三者データ・調査結果で客観性を担保しているか

**障害対処の確認：**
- [ ] 6つの変化の障害に対する応答が準備できているか
- [ ] 抵抗が来た時に論破しようとせず、相手の目標に立ち返れるか
- [ ] 商談後に「安全地帯に戻る」ための対策が講じられているか
