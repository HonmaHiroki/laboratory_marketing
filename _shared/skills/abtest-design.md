# スキル：ABテスト設計テンプレート

**使用するエージェント：** campaign-executor / media-buyer  
**入力：** 改善したい課題・対象チャネル・現状KPI  
**出力：** ABテスト設計書（16項目）・必要サンプルサイズ・ROI試算

---

## 設計手順

### STEP 1：16項目を埋める

```markdown
## ABテスト設計書

| 項目 | 内容 |
|-----|------|
| 課題 | [現在改善したい問題] |
| 仮説 | [課題の原因仮説] |
| 改善案 | [12パターンから選定：下記参照] |
| 成功基準（KPI） | [測定指標＋目標数値＋改善幅] |
| チャネル分類 | オウンド / ペイド / アーンド |
| メディア | [具体的なメディア名] |
| デバイス | PC / タブレット / SP / 全デバイス |
| ページURL | [テスト対象URL] |
| エリア | [ページ内の特定エリア] |
| テスト手法 | 同一URLテスト / スプリットURLテスト / 多変量テスト |
| テストパターン数 | [パターン数] |
| テスト配分 | A:[X]% / B:[X]% |
| 必要サンプルサイズ | [下記STEP 2で算出] |
| 改善ポテンシャル | [下記STEP 3で算出] |
| 改善コスト | 変動費¥X + 固定費（人件費）¥X = 合計¥X |
| ROI | [下記STEP 4で算出] |
```

### STEP 2：必要サンプルサイズを算出する

Sample Size Calculator（https://clincalc.com/stats/samplesize.aspx）で算出する。

設定値：
- Study Group Design：Two independent study groups
- Primary Endpoint：Dichotomous（yes/no）
- Group1 CVR：[現状CVR]
- Group2 CVR：[テストパターン目標CVR]
- Alpha（α）：0.05（有意水準5%）
- Power：0.80（検出力80%）

```
必要サンプルサイズ：各パターン [X] 人
推定テスト期間：[必要サンプル数 ÷ 日次訪問数] 日
```

### STEP 3：改善ポテンシャルを算出する

```
月間改善ポテンシャル =
  月間対象ページ訪問数 × 顧客単価 × CVR改善幅

例：
  月10,000訪問 × ¥3,000 × 0.01（1%改善）= ¥300,000/月
```

### STEP 4：ROIを算出する

```
月次ROI = 月間改善ポテンシャル ÷ 月次コスト × 100

年次ROI = 年間改善ポテンシャル ÷ 総コスト（初期+年間ランニング）× 100
```

### STEP 5：改善案の選択

12パターンから選定する：

| パターン | 内容 | 向いている課題 |
|--------|------|-------------|
| 引き算 | ノイズ要素の削除 | 離脱率が高い |
| 足し算 | 要素の追加 | 情報不足による離脱 |
| 配置替え | レイアウト変更 | CTAが見つからない |
| 要素強調 | 視覚的強調 | 重要情報の見落とし |
| コピーチェンジ | テキスト変更 | CVR改善 |
| ビジュアルチェンジ | 画像変更 | 感情訴求の強化 |
| UIコンポーネントチェンジ | パーツ改修 | UX改善 |
| カラムレイアウト | 段組変更 | 情報密度の調整 |
| 複数要素の最適組み合わせ | 多変量テスト | 複数要因の検証 |
| フロー改修 | ページ遷移変更 | コンバージョンフロー改善 |
| 機能改修 | システム機能変更 | 機能的UX改善 |
| ダイナミックチェンジ | 全面大改修 | 抜本的な見直し |

### STEP 6：テスト手法の選択

| 手法 | 使い所 |
|-----|-------|
| 同一URLテスト | ページ内の一部要素変更（コピー・色） |
| スプリットURLテスト | ページ全体が大きく異なる比較 |
| 多変量テスト | 複数要素の組み合わせ最適化 |
