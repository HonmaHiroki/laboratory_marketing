# Hypothesis Log / Rejected（棄却仮説・失敗分析）

棄却された仮説と失敗の構造分析を保存するフォルダです。
**同じ失敗を繰り返さないために、必ず読むこと。**

## なぜ棄却仮説を残すのか

棄却された仮説は「やってはいけないこと」の地図です。
特に以下のパターンは繰り返されやすいため、明示的に記録します：
- ファン向け施策を未顧客に転用して失敗（ロッサー・リーブスの誤謬）
- ブランドイメージ向上施策が購買に結びつかなかった（因果の取り違え）
- 第一想起を先行指標として追って成長しなかった

## テンプレート（棄却時に追記）

```markdown
## 棄却記録

### 棄却日
YYYY-MM-DD

### 棄却の根拠
（KPI結果と判定基準を具体的に）

### 失敗の構造分析
- 「広告→ブランド」の断絶があったか：Yes / No
- 「文脈→属性」の断絶があったか：Yes / No
- 「広告→店頭」の断絶があったか：Yes / No
- ファン施策を未顧客に誤適用していたか：Yes / No
- 因果の向きを誤っていたか：Yes / No

### なぜ仮説が間違っていたか
（構造的な原因の考察）

### 次回への教訓
（この失敗から何を学ぶか）
```
