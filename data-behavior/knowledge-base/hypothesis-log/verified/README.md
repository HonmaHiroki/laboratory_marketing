# Hypothesis Log / Verified（支持済み仮説：3回未満）

検証で1〜2回支持された仮説を管理します。
3回以上支持されたら `../confirmed/` に移動します。

## ファイル命名規則
`YYYY-MM-DD_[仮説タイトル略称]_verified-[支持回数].md`
（例：`2025-01-15_cep-morning-routine_verified-2.md`）

ファイルの末尾に以下を追記して管理する：

```markdown
## 検証履歴
| 日付 | 施策名 | KPI結果 | 判定 |
|------|-------|---------|------|
| YYYY-MM-DD | [施策名] | [数値] | 支持 |

## 支持回数：X / 3回（次は confirmed/ へ）
## 顧客タイプ別カウント
- ロイヤル顧客での支持：X回
- 未顧客での支持：X回（←成長の源泉として特に重要）
```
